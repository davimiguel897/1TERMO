# Engenharia de Requisitos: Guia de Conteúdo e Técnicas

## 1. Conteúdo de Aula: Requisitos Funcionais vs. Não Funcionais

### Requisitos Funcionais (RF)
* Definem o comportamento esperado do sistema.
* Descrevem as ações que o software deve executar.
* Exemplo: O sistema deve enviar um e-mail de confirmação após o cadastro.

### Requisitos Não Funcionais (RNF)
* Definem as restrições e qualidades do sistema.
* Especificam como o sistema deve realizar as funções.
* Exemplo: O sistema deve processar a requisição em menos de 2 segundos.

---

## 2. Técnicas de Elicitação de Requisitos

### Brainstorm
* Reunião livre para geração rápida de ideias.
* Foco na quantidade de sugestões, sem julgamentos iniciais.
* Ideal para a fase inicial de descoberta do projeto.

### Entrevistas
* Conversas estruturadas ou abertas com stakeholders.
* Coleta de dados aprofundados sobre necessidades específicas.
* Registro direto de expectativas, dores e metas do usuário.

---

## 3. Prototipagem e Modelagem

### Prototipagem
* Criação de modelos visuais da interface do sistema.
* Validação rápida do fluxo com o usuário final.
* Redução de retrabalho antes do desenvolvimento do código.

### Diagramas
* Representação visual da arquitetura e fluxos de dados.
* Uso de Casos de Uso (UML) para mapear atores e ações.
* Facilitação da comunicação entre técnicos e clientes.

---

## 4. Relatórios Técnicos

* Documentação formal das especificações do sistema.
* Registro do Documento de Visão e do backlog do produto.
* Base legal e técnica para homologação e testes de software.
